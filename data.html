MODELING STUFFS

The confiscated assets vocabulary defines a set of domain specific concepts and imports some other vocabularies which can be used in modeling confiscated assets related stuffs.

CONFISCATED ASSETS DATASETS

The confiscated assets datasets ontology reports some some datasets of confiscated assets released by municipalities. One dataset in the catalogue may be confiscated-assets specific or may just contain description of confiscated assets among all the buildings and assets of the municipality. The ontology is available in the following formats:

  • downloadable OWL file;
  • SPARQL endpoint on the Dydra server.

In addition, the catalogue of confiscated assets dataset can be browsed in the datasets page.

Confiscated Assets Ontologies

Some of these datasets have been turned into OWL datasets which uses the confiscated assets vocabulary. For each considered confiscated assets dataset, the following ontologies are provided:

  • assets.owl which contains the descriptions of the confiscated assets in the source dataset;
  • locations.owl in which asset locations are described using the ISA Location Core Vocabulary LOCN;
  • geomerty.owl which contains geospatial information about this locations;
  • log.txt where issues of the source file, such as missing addresses or unrecognizable ones, are reported.

Note that coordinates reported in geometry.owl have been obtained by the google geocoder. As consequence, this information is not freely reusable, but it is subject to the conditions imposed by google.

In addition, these ontologies are provided by a SPARQL endpoint (one for each muncipality).
